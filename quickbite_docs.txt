   1. Project Overview
   Purpose, goals, and what this project demonstrates




1.1 Purpose
QuickBite is a portfolio-grade food delivery mobile app built with Flutter and FastAPI. It is
designed to demonstrate production-level cross-platform development skills to potential clients
on Fiverr and other freelance platforms.
The app simulates a real-world food ordering experience — from browsing restaurants to
tracking an order — using a clean UI, real API integration, and proper state management.



1.2 What This Demonstrates

                      Flutter Skills                                      Backend Skills
 ✓ Complex multi-screen navigation                          ✓ RESTful API design with FastAPI

 ✓ State management with Riverpod                           ✓ Pydantic v2 data validation

 ✓ Custom UI components & theming                           ✓ SQLite/JSON-based data layer

 ✓ Async API integration                                    ✓ CORS, routing, response models

 ✓ Local data persistence (Isar)                            ✓ Modular project structure

3.3 State Management Pattern (Riverpod)

The app uses Riverpod 2.x with the NotifierProvider pattern. Each feature has its own notifier
class that manages business logic, keeping UI widgets completely stateless.


     State Flow
     UI Widget       →     reads Provider → Notifier              → Repository → API / Isar
     UI Widget       ←     rebuilds on state change ←             Notifier emits new state

8.3 Spacing & Layout Rules
     •    Use 8px base grid — all spacing values should be multiples of 8 (8, 16, 24, 32...)
     •    Screen horizontal padding: 16px on all screens
     •    Card border radius: 12px. Button border radius: 8px. Chips: 20px (pill)
     •    Card elevation: use BoxShadow, not Material elevation — more control
     •    Bottom navigation bar height: 64px with SafeArea padding
     •    All images should use BorderRadius.circular(12) with ClipRRect

9.1 Screenshots to Capture
These are the 3 screenshots that should appear in your Fiverr gig image — in order of impact:

 #        Screen                           Why It Works
 1        Home Feed                        First impression — shows cards, images, real data. Clients
                                           immediately recognize this as a real app.
 2        Restaurant Detail + Menu         Shows navigation depth and content-rich UI. Demonstrates you
                                           can handle complex layouts.
 3        Cart or Checkout                 Shows transactional UI — clients building e-commerce or food
                                           apps care about this most.




9.2 Demo Video Tips
A 30–60 second screen recording dramatically increases conversion on Fiverr. Record in this
order:
     •    Open the app — show the splash and home screen loading with shimmer, then data
          appearing
     •    Tap a restaurant — show the smooth navigation transition and menu loading

Before sharing the repo link in your Fiverr gig, make sure:
